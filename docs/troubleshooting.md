# Troubleshooting - Решение проблем

> Руководство по устранению типичных проблем в Trading Platform

---

## Данные по сделкам/позициям не поступают или отображаются с задержкой

### Возможные причины и решения:

**1. Проблемы с подключением к торговому серверу (MT4/MT5)**
- Проверьте статус подключения в разделе **Nodes** → убедитесь что нода показывает статус "Connected"
- Проверьте правильность учетных данных Manager API в настройках ноды
- Убедитесь что IP-адрес Trading Platform добавлен в whitelist на торговом сервере

**2. Проблемы с Manager API правами**
- Откройте MT4/MT5 Manager и проверьте права пользователя, используемого Trading Platform
- Необходимые права: чтение счетов, чтение сделок, чтение позиций
- Для некоторых функций нужны расширенные права (например, для CID Control)

**3. Задержка синхронизации данных**
- Trading Platform синхронизирует данные с интервалом (обычно 1-5 секунд)
- При большом количестве счетов или позиций синхронизация может занимать больше времени
- Проверьте нагрузку на сервер Trading Platform в логах

**4. Фильтрация данных**
- Проверьте настроенные фильтры в текущем разделе (группы, ноды, временные периоды)
- Убедитесь что выбраны правильные ноды в фильтре
- Сбросьте фильтры и проверьте данные снова

**5. Проблемы с кэшированием браузера**
- Обновите страницу (F5 или Ctrl+R)
- Очистите кэш браузера (Ctrl+Shift+Del)
- Попробуйте открыть в режиме инкогнито

### Диагностика:

1. **Проверка статуса нод:**
   - Перейдите в **Settings** → **Nodes**
   - Убедитесь что все ноды имеют статус "Connected" (зеленый индикатор)

2. **Проверка логов:**
   - Просмотрите логи Trading Platform на наличие ошибок подключения
   - Обратите внимание на сообщения типа "Connection lost" или "Authentication failed"

3. **Проверка данных напрямую:**
   - Откройте MT4/MT5 Manager и проверьте наличие данных там
   - Если данные есть в Manager но нет в Trading Platform - проблема на стороне Trading Platform

### Если проблема не решена:

Свяжитесь с технической поддержкой, предоставив:
- Скриншоты ошибок
- Название ноды с проблемой
- Время возникновения проблемы
- Логи Trading Platform (если доступны)

## Не отображаются счета или группы

### Проверьте:

1. **Права доступа пользователя Trading Platform:**
   - Перейдите в **Settings** → **Users**
   - Убедитесь что у вашего пользователя есть права на просмотр нужных групп

2. **Настройки видимости групп:**
   - Некоторые группы могут быть скрыты в настройках
   - Проверьте **Settings** → **Groups Settings**

3. **Фильтр по нодам:**
   - Убедитесь что выбрана правильная нода в фильтре

## Неправильный расчет P&L или баланса

### Возможные причины:

1. **Разница во времени закрытия сессии:**
   - Trading Platform использует UTC для расчетов
   - Торговый сервер может использовать другой часовой пояс
   - Проверьте настройки времени в **Settings** → **Trade Session**

2. **Неучтенные комиссии или свопы:**
   - Убедитесь что настройки комиссий в Trading Platform соответствуют настройкам на торговом сервере

3. **Кэш данных:**
   - Попробуйте обновить данные вручную или подождите следующую синхронизацию

## Триггеры не срабатывают

### Проверьте:

1. **Активирован ли триггер:**
   - Откройте настройки триггера
   - Убедитесь что переключатель "Active" включен

2. **Правильно ли настроены условия:**
   - Проверьте пороговые значения
   - Убедитесь что выбраны правильные группы/счета

3. **Есть ли необходимые права:**
   - Некоторые триггеры требуют специфических прав Manager API
   - См. документацию по конкретному триггеру

4. **Проверьте журнал триггеров:**
   - **Risk Management** → **Triggers Log**
   - Посмотрите срабатывали ли триггеры ранее

## Алерты и уведомления по превышению лимитов

Trading Platform предоставляет несколько типов триггеров для мониторинга превышения лимитов:

### Алерты по объему (Volume Alerts)

**Large Volume by Account** - мониторинг общего объема позиций на счете:
- **Risk Management** → **Anti-Fraud System** → **Large Volume by Account**
- Настройка: укажите минимальный объем (в лотах или USD) для срабатывания алерта
- Триггер создаст уведомление когда суммарный объем открытых позиций превысит порог

**Large Volume by Order** - мониторинг объема отдельных ордеров:
- **Risk Management** → **Anti-Fraud System** → **Large Volume by Order**
- Сработает при открытии ордера с объемом выше заданного порога

### Алерты по плечу (Leverage Alerts)

**Top Margin** - мониторинг счетов с высоким уровнем маржи:
- **Dealing Desk** → **Top Margin**
- Отображает счета с наибольшим торговым плечом в реальном времени
- Это раздел отчетности, не триггер

**Leverage by Equity** - автоматическое управление плечом:
- **Risk Management** → **Automatic Control** → **Leverage by Equity**
- Автоматически понижает плечо при росте эквити выше заданных порогов
- Может отправлять уведомления при изменении плеча

**Dynamic Leverage** - динамическое плечо по объему:
- **Risk Management** → **Automatic Control** → **Dynamic Leverage**
- Повышает маржинальные требования при увеличении объема позиций

### Алерты по убыткам (Loss Alerts)

**Winners & Losers** - мониторинг прибыльных и убыточных счетов:
- **Dealing Desk** → **Winners & Losers**
- Отображает топ убыточных счетов за период
- Фильтрация по дню, неделе, месяцу

**Margin Calls** - счета приближающиеся к stop-out:
- **Dealing Desk** → **Margin Calls**
- Показывает счета с низким уровнем маржи в реальном времени

**Achieved Profit** - достижение процента профита:
- **Risk Management** → **Anti-Fraud System** → **Achieved Profit**
- Уведомляет когда трейдер достигает заданного % прибыли от депозита

### Настройка уведомлений

Все триггеры отправляют уведомления в:
1. **Раздел Risk Management** - список активных нотификаций
2. **Иконка молнии** ⚡ в правом верхнем углу интерфейса
3. **Email/Telegram** - при настройке интеграций в Settings

## A-Book vs B-Book - анализ стоимости

### Что такое A-Book и B-Book?

**A-Book (STP/ECN модель):**
- Ордера клиентов выводятся на внешнего провайдера ликвидности
- Брокер зарабатывает на комиссиях и спредах
- Риск брокера минимален - убытки клиента не = прибыль брокера

**B-Book (Market Maker модель):**
- Ордера исполняются внутри брокера
- Брокер является контрагентом сделки
- Прибыль клиента = убыток брокера и наоборот

### Анализ стоимости и распределения

**Раздел A/B Books Net:**
- **Dealing Desk** → **A/B Books Net**
- Показывает соотношение объемов A-Book и B-Book по каждому символу
- Столбец **Comparison, %** - процент A-Book относительно B-Book

**Настройка A-Book:**
- **Dealing Desk** → **A-Book Settings**
- Определение какие группы/счета относятся к A-Book
- Настройка % покрытия (частичное хеджирование)

### Критерии распределения по букам

Рекомендации для A-Book (вывод на ликвидность):
- Трейдеры со стабильной прибыльностью
- Крупные объемы сделок
- Профессиональные трейдеры

Рекомендации для B-Book:
- Новички с небольшими депозитами
- Трейдеры со статистикой убытков
- Мелкие сделки (cost of hedging > potential loss)

### Расчет стоимости хеджирования

Стоимость A-Book для брокера:
- Комиссия LP (провайдера ликвидности)
- Спред LP (обычно шире чем показывается клиенту)
- Свопы на overnight позиции
- Технические расходы на интеграцию

При решении о переводе счета на A-Book, сравните:
- Ожидаемую прибыль трейдера
- Стоимость хеджирования этой прибыли

## Hedge не закрылся - проблемы с хеджированием

### Возможные причины незакрытия хеджа:

**1. Проблемы с ликвидностью на LP:**
- Недостаточная ликвидность для закрытия позиции
- LP отклонил ордер на закрытие
- Проверьте логи LP-подключения

**2. Проблемы с правами Manager API:**
- Недостаточно прав для торговых операций
- Проверьте что у пользователя API есть права на торговлю

**3. Технические проблемы:**
- Разрыв соединения в момент закрытия
- Таймаут при исполнении
- Ошибка синхронизации между MT и LP

**4. Проблемы с настройкой триггера:**
- Хедж был открыт но условие закрытия не сработало
- Проверьте настройки триггера который должен был закрыть позицию

### Диагностика:

1. **Проверьте историю ордеров на LP:**
   - Был ли отправлен запрос на закрытие?
   - Какой статус у запроса?

2. **Проверьте логи Trading Platform:**
   - Поиск ошибок связанных с hedge closure
   - Проверка сообщений об отклоненных ордерах

3. **Проверьте состояние позиции на MT сервере:**
   - Позиция действительно открыта?
   - Нет ли противоположной позиции которая компенсирует?

### Ручное закрытие:

Если автоматическое закрытие не сработало:
1. Откройте MT Manager
2. Найдите открытую hedge-позицию
3. Закройте позицию вручную
4. Проверьте настройки автоматизации для предотвращения повторения

## Расхождение между Net-Volume - причины расхождений

### Что такое Net-Volume?

Net-Volume = Buy Volume - Sell Volume (чистый объем позиции по символу)

### Частые причины расхождений:

**1. Разница в расчете между секциями:**
- **Net Summary** показывает текущие открытые позиции
- **Reporting** может показывать исторические данные
- Убедитесь что смотрите данные за один период

**2. Фильтры:**
- Разные фильтры применены в разных секциях
- Проверьте настройки фильтров (ноды, группы, символы)

**3. Collect Symbol Suffixes:**
- Если включено - EURUSD, EURUSDc, EURUSDm объединяются
- Если выключено - каждый суффикс считается отдельно
- Проверьте что настройка одинакова в сравниваемых разделах

**4. Dealer Side toggle:**
- ON = объемы с точки зрения дилера
- OFF = объемы с точки зрения трейдера
- Buy и Sell меняются местами при переключении

**5. Разница во времени:**
- Данные могут отображаться с небольшой задержкой
- Real-time vs cached data

**6. A-Book vs B-Book фильтрация:**
- A/B Books Net показывает только данные настроенных буков
- Net Summary может показывать все позиции

### Как проверить:

1. Откройте Net Summary и A/B Books Net в отдельных вкладках
2. Примените одинаковые фильтры
3. Убедитесь что Collect Symbol Suffixes одинаков
4. Сравните данные по конкретному символу

## Расхождение между Equity и Exposure - объяснение различий

### Что такое Equity и Exposure?

**Equity (эквити):**
- Balance + Floating P&L + Swaps + Commissions
- Реальные средства доступные клиенту
- Меняется с каждым тиком цены

**Exposure (экспозиция):**
- Общий объем открытых позиций в валютном эквиваленте
- Показывает рыночный риск брокера
- Рассчитывается как Volume × Contract Size × Rate

### Почему они могут расходиться?

**1. Это разные метрики:**
- Equity - деньги клиента
- Exposure - объем рисковых позиций
- Они НЕ должны быть равны

**2. Пример расхождения:**
- Клиент с Equity $10,000
- Открывает позицию 1 лот EURUSD
- Exposure = 100,000 USD (contract size)
- Equity всё ещё около $10,000

**3. Actual Leverage связывает их:**
- Actual Leverage = Net Volume / Equity
- Если Equity = $10,000 и Net Volume = $100,000
- Actual Leverage = 1:10

### Где смотреть:

- **Equity по счетам:** Dealing Desk → Top Clients, Account Information
- **Exposure по символам:** Dealing Desk → Exposures
- **Связь между ними:** колонка Actual Leverage в различных отчетах

### Типичные вопросы:

**Q: Почему Exposure больше Equity?**
A: Это нормально при использовании плеча. Exposure = объем позиций, Equity = реальные деньги.

**Q: Equity отрицательная, а Exposure положительная?**
A: Счет в margin call / stop out. Позиции открыты, но equity ушла в минус из-за убытков.

## Почему увеличился cumulative loss при нулевом Net Volume (P&L Report Discrepancy)

### Вопрос: "Why did the cumulative loss on GBPUSD increase in the P&L report, even though the net volume remained at zero?"

Это один из частых вопросов по P&L отчетам. Объяснение:

### Причины увеличения cumulative loss при нулевом Net Volume:

**1. Net Volume = 0 не означает отсутствие торговли**
- Net Volume = Buy Volume - Sell Volume
- Если клиент открыл 1 лот Buy и 1 лот Sell = Net Volume = 0
- Но обе позиции могут генерировать убыток!

**2. Свопы (Swaps)**
- Overnight свопы начисляются на каждую позицию отдельно
- Даже при нулевом Net Volume свопы продолжают накапливаться
- Особенно заметно на парах с высоким swap rate

**3. Спред при открытии/закрытии**
- Каждая сделка теряет спред при входе
- При частой торговле (даже с нулевым Net Volume) потери на спреде накапливаются

**4. Комиссии**
- Комиссии списываются при каждой сделке
- Высокая торговая активность = высокие комиссии = рост cumulative loss

**5. Закрытые убыточные сделки**
- Net Volume показывает ТЕКУЩИЕ открытые позиции
- Cumulative Loss включает ВСЕ закрытые убыточные сделки за период
- Клиент мог закрыть много убыточных позиций, но Net Volume остался на нуле

### Пример:

День 1: Открыт Buy 1 лот GBPUSD → Net Volume = +1
День 2: Закрыт Buy с убытком -$500, открыт Sell 1 лот → Net Volume = -1
День 3: Закрыт Sell с убытком -$300, открыт Buy 1 лот → Net Volume = +1
День 4: Закрыт Buy с убытком -$200 → Net Volume = 0

**Итого:** Net Volume = 0, но Cumulative Loss = $1000

### Где смотреть детали:

1. **Current Session** → P&L breakdown по символам
2. **Winners & Losers** → история прибыльных/убыточных сделок
3. **Account Information** → вкладка History для конкретного счета
4. **Reporting** → детальные отчеты с разбивкой по сделкам

## Формулы и расчеты в Trading Platform

### Основные формулы:

**Equity:**
```
Equity = Balance + Floating P&L + Swaps + Commissions
```

**Margin Level:**
```
Margin Level = (Equity / Margin) × 100%
```

**Actual Leverage:**
```
Actual Leverage = Net Volume / Equity
```

**Net Volume:**
```
Net Volume = Σ(Buy Volume) - Σ(Sell Volume)
```

**Drawdown:**
```
Drawdown = (Peak Equity - Current Equity) / Peak Equity × 100%
```

**VWAP (Volume Weighted Average Price):**
```
VWAP = Σ(Price × Volume) / Σ(Volume)
```

Подробнее см. документацию "Формулы и расчеты"

- Сравнение чистых объемов A-Book и B-Book
- Столбец **Volume In USD** - чистый объем в долларах
- График справа показывает соотношение A/B

**3. Dealing Desk → Exposures**
- Чистая экспозиция по валютам (не по символам)
- Показывает общий риск по каждой валюте

**4. Dashboards → Big Picture Dashboard**
- Виджет "Net exposure by symbols"
- Визуализация чистой экспозиции

**5. Dashboards → Geo Net Volumes Dashboard**
- Чистые объемы с разбивкой по географии
- Виджеты: Traders vs Robots, B-Book vs A-Book

### Группировка по символам:

Опция **Collect Symbol Suffixes**:
- **ON**: EURUSD + EURUSDc + EURUSDm → объединяются как EURUSD
- **OFF**: каждый суффикс отдельно

## Как просмотреть статистику P&L по клиентам, активам, группам

### P&L в реальном времени:

**1. Current Session** (главный раздел для сессионного P&L)
- Графики P&L в реальном времени
- Разбивка по:
  - **Session P&L by account groups** - P&L по группам счетов
  - **Session P&L by symbol groups** - P&L по группам символов
  - **Session P&L by trading reasons** - P&L по причинам торговли
- Элементы управления для сортировки данных

**2. Dealing Desk → Winners & Losers**
- Топ прибыльных счетов (Winners) слева
- Топ убыточных счетов (Losers) справа
- Фильтрация по периоду: Daily, Weekly, Monthly, All-time

**3. Dealing Desk → Top Clients**
- Столбцы Session PnL USD, Total PnL USD
- Сортировка по любой метрике

### Исторический P&L:

**1. Dashboards → Total P&L & Winners**
- Общий P&L за месяц, неделю, день
- Прибыльность по группам счетов
- Прибыльность по причинам торговли

**2. Reporting → Reports**
- Детальные исторические отчеты
- Экспорт в CSV/Excel
- Кастомные фильтры и периоды

**3. Reporting → Metabase**
- SQL запросы для продвинутого анализа
- Создание кастомных дашбордов
